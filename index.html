ng-polymer-elements

AngularJS two-way binding for Web Components

Out of the box support for Polymer Core and Paper elements

Why do I need it?

AngularJS and Web Components both provide binding mechanisms, and for some cases you can bind the scope properties to Web Components attributes. For example, the following code works fine: 

$scope.buttonLabel = 'Click me';

$scope.buttonAction = function() {
   alert('Button clicked');
};
<paper-button label="{{buttonLabel}}" on-click="buttonAction()">
</paper-button>

There are two cases where you can't get Web Components to work with your AngularJS models:

  • Having Web Components update your model, for example <core-input value="{{myModel}}"></core-input> will not change myModel when the user changes the input text.
  • Binding anything other than a string, for example <core-list data="{{myList}}"><core-list> will not work because the expression {{myList}} will be treated as a string.
Enters ng-polymer-elements.

What does it do?

ng-polymer-elements creates directives for existing custom elements, allowing you to use them as you would any directive. The following example shows how you can use core-input and paper-input with a model as if they were "plain" input: 

<input type="text" ng-model="text"/>
<core-input ng-model="text"></core-input>
<paper-input ng-model="text""></paper-input>

How do I install it?

Either install it with bower using bower install ng-polymer-elements or download the release.

Add ng-polymer-elements to your module's dependencies.

AngularJS should be bootstrapped only after the custom elements have been registered. If you use automatic bootstrap (ie. <html ng-app="myModule">), ng-polymer-elements takes care of this, but if you use manual bootstrap, you need to call angular.bootstrap() only after Polymer has been loaded, and with a wrapped DOM element: 

window.addEventListener('polymer-ready', function() {
   angular.bootstrap(wrap(document), ['myModule']);
});

Also remember to include the Polymer platform script and any imports you use in your HTML head, and to add the unresolved attribute to your HTML body. The general structure of your HTML should be like so: 

<!DOCTYPE html>
<html ng-app="myModule">
   <head>
      <script src="platform.js"></script>
      <!-- your imports here -->
   </head>
   <body unresolved>
      <!-- your view here -->
      <script src="angular.js"></script>
      <script src="ng-polymer-elements.js"></script>
      <!-- your scripts here -->
   </body>
</html>

We saw above how to bind the value of core-input and paper-input. Here is a similar example, this time with multiline:

<textarea ng-model="multilineText"></textarea>
<core-input ng-model="multilineText" multiline></core-input>
<paper-input ng-model="multilineText" multiline></paper-input>

The following example shows the binding of a paper-radio-group selected value:

<paper-radio-group ng-model="radio">

   <paper-radio-button name="two" label="Two"></paper-radio-button>
   <paper-radio-button name="three" label="Three"></paper-radio-button>
</paper-radio-group>

The following example shows the binding of paper-checkbox and paper-toggle-button to a boolean value:

<paper-checkbox ng-model="boolean"></paper-checkbox>

Boolean values can be used to toggle core-overlay, paper-dialog and paper-toast on and off. The two way binding means that hiding these elements will also set the bound value to false.

<paper-button label="Dialog" ng-click="dialog=true"></paper-button>
<span>Dialog opened: {{dialog}}</span>
<paper-dialog ng-opened="dialog" heading="Paper Dialog" layered>
	<!-- dialog content -->
</paper-dialog>

<paper-button label="Toast" ng-click="toast=true"></paper-button>
<span>Toast opened: {{toast}}</span>
<paper-toast ng-opened="toast" text="This is a toast"></paper-toast>
Dialog opened: {{dialog}}

Some content goes here
Toast opened: {{toast}}

Support for core-list allows you to bind an array of items to be shown, and to bind an event to get the item the user clicked/tapped on. Add the $event parameter to your event handler call and get the selected item from $event.detail.data:

<core-list ng-model="list" ng-tap="onSelect($event)">
   <template>
      <div>{{name}}</div>
   </template>
</core-list>
$scope.list = [{name:'One'},{name:'Two'},{name:'Three'}];

$scope.onSelect = function(event) {
   var selected = event.detail.data;
   alert('Selected: ' + selected.name);
};

How do I use it with other custom elements?

ng-polymer-elements uses a simple structure to map directive attributes to custom element properties. You can extend the mapping by setting window.NG_POLYMER_ELEMENTS_EXTENDED_MAPPINGS with your own mappings before ng-polymer-elements[.min].js is invoked.

The mapping structure is an object where the keys are the directive names, and their values are objects that contain a key for each directive attribute you want to map. The values for the directives are objects containing a single entry where the key is the type (primitive, array, object or event) and the value is the custom element property.

Consider the mappings for core-input and core-list:

window.NG_POLYMER_ELEMENTS_EXTENDED_MAPPINGS = {
   coreInput: {
      ngModel: {
         primitive: 'inputValue'
      }
   },
   coreList: {
      ngModel: {
         array: 'data'
      },
      ngTap: {
         event: 'core-activate'
      }
   }
};

For core-input (coreInput) we map the ng-model attribute to the inputValue property as a primitive (string in this case). For core-list (coreList) we map the ng-model attribute to the data property as an array, and the ng-tap attribute to the core-activated event. when mapping events, the original event will be available to AngularJS handlers as the $event parameter.
